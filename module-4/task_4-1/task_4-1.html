1. Why is object-oriented development so important in programming?

    Objektorientert utvikling (OOP) er viktig fordi den gjør koden strukturert, gjenbrukbar og lettere å vedlikeholde. I stedet for å skrive samme kode mange ganger, kan man lage klasser og objekter som kan brukes flere steder. Det gjør det enklere å bygge store programmer, feilsøke, og videreutvikle systemer uten å ødelegge resten av koden. Du deler koden inn i små, logiske deler (objekter) Hver del har sitt ansvar (f.eks. en konto, en kunde, en bil) Du får orden, gjenbruk og mindre feil

  2. In Task 1, we create a static object called "AccountType." Explain what this object is used for and how this type of object can help us in software development.

    AccountType er et statisk objekt som inneholder konstanter (faste verdier) for kontotyper, som f.eks. const AccountType = { Normal: "Brukskonto", Saving: "Sparekonto", Pension: "Pensjonskonto" }; Dette objektet brukes for å slippe skrive inn strenger manuelt overalt i koden. I stedet for å skrive "Sparekonto" hver gang, kan vi bruke AccountType.Saving. Dette hjelper oss fordi: Det reduserer feil (ingen skrivefeil i tekst) Det gjør koden ryddigere og enklere å lese Det er lettere å endre senere (du endrer bare ett sted)

  3. Discuss why we use encapsulation in classes. Why do we want to "hide" code within our class? Explain how we can create "private" and "public" members in a constructor function

    Innkapsling betyr at vi skjuler data og funksjoner inni klassen, slik at de ikke kan endres direkte utenfra. Vi vil «gjemme» deler av koden for å beskytte dataene og sørge for at alt skjer på riktig måte. Private medlemmer: lages med #, f.eks. #balance (kan bare brukes inni klassen). Public medlemmer: vanlige navn uten #, f.eks. deposit() (kan brukes utenfra).
