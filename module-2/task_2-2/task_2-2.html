1. Where should you use constants, and where should you use variables? Can they all be constants?

    2. - Du bruker const når når verdien aldri skal endres for eksempel pi og valutakurs. Når du bruker let er det når verdien skal endres underveis for eksempel teller og input fra brukeren. Det er ikke alt som kan være konstant, siden programmene trenger ofte verdier som skal oppdateres.

  2. How do parentheses affect the arithmetic outcome of an expression?

    3. - De bestemmer rekkefølgen på beregninger. Uten parenteser brukes matematikkens standardregler (gang/div før pluus/minus). Parenteser kan endre utfallet ved å tvinge en annen rekkefølge.

  3. How do we limit floating-point numbers to a given number of decimals?

    4. - Vi kan bruke .toFixed(n) for å vise n desimaler

  4. How can we compare two floating-point values to see if they are equal (is 12.199999 the same as 12.2)?

    5. Vi kan ikke sammenligne flyttall direkte med ===, fordi små avrundingsfeil kan gjøre at tall som ser like ut, ikke blir helt like i datamaskinen. I stedet sammenligner vi forskjellen mellom tallene med en veldig liten verdi (epsilon). Hvis forskjellen er mindre enn denne verdien, regner vi tallene som like.

  5. What do comparison operators return, and how can we check if they are FALSE or TRUE?

    7. - Det gir et tilfeldig tall mellom 0 og 1. Du kan bruke det til å lage tilfeldige tall i et område for eksempel terningkast eller og simuleringer

  7. What is modulus (%), and when do we need this operator in mathematical operations?
